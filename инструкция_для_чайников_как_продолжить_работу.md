# Инструкция для чайников: как продолжить работу с файлами репозитория

## Короткий ответ
- **Лучший вариант для долгой работы:** держать всё в GitHub-репозитории и синхронизировать локальную копию перед каждой сессией.
- **Загрузка файлов в чат** подходит для разовой проверки/правки, но хуже для большой и длительной разработки.

---

## Вариант 1 (рекомендуется): работать через GitHub

### Шаг 1. Проверьте, что репозиторий подключен
В папке проекта выполните:

```bash
git remote -v
```

Если пусто — добавьте `origin`:

```bash
git remote add origin https://github.com/builovmaikl-art/SmartHome.git
```

### Шаг 2. Подтяните актуальные файлы

```bash
git fetch origin
git checkout main
git pull --ff-only origin main
```

### Шаг 3. Убедитесь, что нужные файлы реально появились

```bash
find . -maxdepth 2 -type f | sed 's#^./##' | sort
```

Проверьте, что есть:
- `ТЗ .txt`
- `Система умного дома.export`
- `result.txt`

### Шаг 4. Продолжайте работу в новой ветке

```bash
git checkout -b work-next-step
```

После правок:

```bash
git add .
git commit -m "..."
git push -u origin work-next-step
```

---

## Вариант 2: загружать файлы прямо в чат

Используйте этот вариант, если:
- нужно быстро посмотреть/исправить 1–3 файла;
- нет доступа к git на текущем устройстве;
- это разовая задача.

Минусы:
- легко потерять связь между версиями;
- сложнее вести историю изменений;
- сложнее работать с большими файлами и несколькими итерациями.

---

## Что выбрать именно вам

С учетом вашей задачи (ТЗ, export-файл, много итераций, архитектурные правки):
- **лучше GitHub-подход (Вариант 1)**;
- в чат загружать только то, чего временно нет в локальной копии.

---

## Если снова “файл есть на GitHub, но не виден в чате”

Сделайте мини-чеклист:

1. `git remote -v` — есть ли `origin`?
2. `git pull --ff-only origin main` — точно обновились?
3. `find ... | sort` — файл физически есть в рабочей папке?
4. Если нет — проверьте, не включен ли sparse-checkout:

```bash
git sparse-checkout list
git sparse-checkout disable
```

5. Повторите `git pull`.

---

## Практический совет
- Перед каждой новой сессией всегда делайте:

```bash
git fetch origin && git pull --ff-only origin main
```

Это экономит много времени и убирает 90% проблем “почему ChatGPT не видит файл”.
